Abstract factory


Scopo

Chiamando una factory insieme ad un valore usato come input sarà restituito un oggetto a seconda di questo input

Code

//Creazione di un interface, obbligo a qualunque classe lo implementa di averlo.
package abstractfactory;
public interface Game {
    public void print();
}

//Main, crea un oggetto "Game", questo oggetto avrà per forza il metodo Print(),
// per questo sarà utilizabile.
package abstractfactory;
public class AbstractFactory {
    public static void main(String[] args) {
        
        Game X = GameFactory.CreateGame("Lego");
        X.print();
    }    
}

// Creazione di un giocattolo Robot
package abstractfactory;
public class Robot implements Game{
    @Override
    public void print() {System.out.println("Robot!");}
}

// Creazione di un giocattolo Lego
package abstractfactory;
public class Lego implements Game{
    @Override
    public void print() {System.out.println("Lego!");}
}

// L'Abstract factory, una classe abstract quindi non inizializabile, 
// quello usabile è il metodo, che restituirà a seconda del input un diverso oggetto. 
package abstractfactory;
public abstract class GameFactory {
    public static Game CreateGame(String Game)
    {
        if(Game == "Lego"){return new Lego();}
        else if(Game == "Robot"){return new Robot();}
        return null;
    }
}

Come

Partiamo da qualche presupposto: Un interfaccia obbliga una classe che la implementa ad avere quei metodi per forza, invece una classe abstract non permette la inizializazione di quella classe, possiamo accedere ai metodi se resi statici.
Sapendo questo possiamo creare la "abstract factory"...
Si chiama abstract factory difatti perchè la classe "fabbrica" che l'utilizza è "abstract", cioè non si può inizializare, ma ha dei metodi statici che ritornano un oggetto che implementa un interfaccia... Questa interfaccia nel main verrà richiamata e l'interfaccia che è stata resituita ha i metodi che sono stati obbligatoriamente implementati dall'interfaccia, che sono utilizabili.
